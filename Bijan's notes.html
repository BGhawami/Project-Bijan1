Lesson 1: Basics

What a web page is?

A web page is a text document written in a language called HTML Web browsers read these documents, and then interpret and display them.

How coding works: Coding happens when programmers write text in a language that a computer can understand.The computer can then follow the instructions the programmer wrote.

Computers are stupid. Therefore write with the correct syntax!

Programmers can't remember everything - that is okay; you need to know where to look it up

Tags and elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags An HTML tag is always contained within angled brackets. Most tag have an opening tag and a closing tag.ELEMENT = An HTML element refers to every thing within a set of opening and closing tags.ATTRIBUTE = This is a property of an HTML element. For example to set the href attribute of an anchor tag.

Why Computers are Stupid

Computers are stupid because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Structure of a page HTML - Hyper Text Markup Language - the structured markup language to create webpages. CSS - Cascading style shetts - style sheet language used for describing the look and formatting of a document within an markup language.DOM -Document Object Model - a cross platform and language independent convention for representing and interacting with objects in HTML and other markup languages.The nodes of every document are organzised in a tree structure, called the DOM tree.

From a programmers perspective, everything on a page is a box; "boxify a page". The weblayout is scetched out in boxes

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree.

In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier.

Lesson 3: Layout

Styling and CSS

Search think about CSS as search and replace tool, identify a class, id or tag of the element you want ot find and then replace with properties, values you want.

  • Order in CSS matters: define rules in the order you want them applied
  • Default style of a browser
  • Apply styles through a seperate linked css file- Best method to work with.
  • Creating your styles within the HTML document, not regularly used, in practice works with small projects that don't require extensive changes and styling.
  • Inline styling within the individual element, not effective or efficient as it will cause repetition which is what programing tries to avoid.

Specifics or specificity matters - "Cascading" means that rules are applied not only to the elements they directly match, but also to all those elements child elements. However; if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

All webpages are built with the box model in mind, breaking all the individual elements into boxes, in order to better organize the structure of the page.

Steps of the web programmer:
  1. Look for natural boxes
  2. look for repeated styles & semantic elements.
  3. Write your HTML
  4. Apply Styles (from the biggest to the smallest - remember specificity
  5. Tweak and fix as you repeat steps 3 - and 4

* also test in all major browsers.
